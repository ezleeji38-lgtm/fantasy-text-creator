# Revision & Rewrite Engine (리라이팅 엔진)

판타지 소설 장편 원고의 리라이팅을 체계적으로 수행하는 도구. 장편 수정의 핵심 도전인 연쇄 영향을 관리하는 데 특화되어 있다.

핵심 원칙: 장편 판타지에서 한 곳을 고치면 다른 곳이 깨진다. 3장의 마법 규칙을 바꾸면 12장의 전투가 논리적으로 작동하지 않게 되고, 주인공의 동기를 바꾸면 5장과 15장의 대화가 어색해진다. 이 스킬은 수정의 파급 효과를 미리 추적하고, 올바른 순서로 수정을 진행하는 계획을 수립한다.

---

## 리라이팅 프로세스

### 단계 1: 진단 수집

수정이 필요한 문제를 수집한다. 입력 원천:
- beta-reader-simulator의 피드백 리포트 (독자 관점의 문제)
- consistency-auditor의 일관성 오류 목록 (기술적 오류)
- pacing-rhythm-optimizer의 페이싱 진단 (리듬 문제)
- 작가(사용자) 본인의 수정 노트 (창작 의도에 기반한 변경)

### 단계 2: 수정 계층 분류

수집된 문제를 5단계 계층으로 분류한다. 위에서 아래로 수정해야 하위 단계의 작업이 낭비되지 않는다:

```
Level 1 구조 수정:
  플롯 구조 변경, 시퀀스 재배치, 서브플롯 추가/삭제
  → 가장 먼저 수정. 구조가 바뀌면 나머지 모든 것이 영향받음.

Level 2 장면 수정:
  장면 추가/삭제/이동, 장면 내 핵심 사건 변경
  → 구조 확정 후 수정.

Level 3 캐릭터/관계 수정:
  캐릭터 동기 변경, 관계 조정, 아크 수정, 대사 수정
  → 장면 확정 후 수정.

Level 4 문체 수정:
  문장 다듬기, 묘사 추가/삭제, 톤 조정, Info-dump 해소
  → 내용 확정 후 수정.

Level 5 교정:
  맞춤법, 문법, 고유 명사 표기 통일, 형식 정리
  → 최종 단계.
```

이 계층을 무시하고 Level 4(문체)부터 다듬으면, Level 1(구조) 변경 시 다듬은 문장이 삭제되는 낭비가 발생한다.

### 단계 3: 연쇄 영향 추적

각 수정 항목에 대해 영향받는 다른 부분을 추적한다:

```
수정: "3장에서 마법 비용을 '체력 소모'에서 '기억 상실'로 변경"

연쇄 영향:
  → ch.7: 주인공이 마법 사용 후 피로해하는 장면
    → 조치: 피로 묘사를 기억 혼란 묘사로 변경
  → ch.12: 전투에서 마법 연속 사용
    → 조치: 기억 상실의 누적 효과 반영
  → ch.15: 주인공이 과거를 회상하는 장면
    → 조치: 기억 상실로 인한 불완전한 회상으로 재설계
  → ch.18: 마법 비용에 대한 대화
    → 조치: 대사 전면 수정
  → 캐릭터 아크: "기억"이 주제적 무게를 가지게 됨
    → 조치: 아크와의 통합 검토 (기회가 될 수 있음)
```

### 단계 4: 수정 실행 계획

수정 순서, 영향 범위, 예상 작업량을 포함한 실행 계획을 수립한다:

```
수정 실행 계획
━━━━━━━━━━

Round 1 (구조 수정):
  - 수정 A: [내용] → 영향: ch.X, ch.Y, ch.Z
  - 수정 B: [내용] → 영향: ch.P, ch.Q
  → Round 1 완료 후 consistency-auditor 빠른 점검

Round 2 (장면 수정):
  - 수정 C: [내용] → 영향: 해당 장면 내
  → Round 2 완료 후 pacing-rhythm-optimizer 재점검

Round 3 (캐릭터/관계 수정):
  → Round 3 완료 후 대화 일관성 점검

Round 4 (문체 수정):
  → Round 4 완료 후 prose-style-engine 진단

Round 5 (교정):
  → consistency-auditor 최종 실행
```

### 단계 5: 수정 후 검증

수정 완료 후:
- consistency-auditor로 일관성 재검증 (특히 연쇄 영향 지점)
- pacing-rhythm-optimizer로 페이싱 재검증
- beta-reader-simulator로 수정 효과 확인 (선택: 핵심 장면만)

## 수정 유형별 가이드

### 구조 수정 (Level 1)

구조 수정은 가장 파급력이 크다. 수정 전에 전체 비트 시트(plot-architecture-engine)를 재점검하고, 변경 후의 새 비트 시트를 먼저 작성한 후 실제 원고 수정에 들어간다.

주요 구조 수정 유형:
- 시퀀스 재배치: 챕터/장면의 순서 변경
- 서브플롯 수술: 서브플롯 추가, 삭제, 또는 메인 플롯과의 연결 강화
- 전환점 이동: 핵심 전환점의 구조적 위치 변경
- 시작점 변경: In medias res로 시작점을 옮기거나, 프롤로그 추가/삭제
- POV 변경: 시점 인물의 추가/삭제/교체

### 장면 수정 (Level 2)

장면 수정은 개별 장면의 내부를 변경한다:
- 장면 목적 재설정: 기능이 약한 장면에 새 기능 부여
- 장면 분할/합치기: 과도하게 긴 장면 분할, 약한 장면들 합치기
- 결과 변경: 장면의 결과를 "예"에서 "예, 그러나"로 변경하여 긴장 증폭

### 캐릭터 수정 (Level 3)

캐릭터 수정은 character-arc-engineer의 프로파일을 참조한다:
- 동기 조정: 캐릭터의 Want 또는 Need 변경
- 아크 강화: 거짓 믿음의 도전이 더 선명하게 드러나도록
- 관계 조정: relationship-web-mapper의 관계 변화 곡선 수정

### 문체 수정 (Level 4)

prose-style-engine의 진단 모드 결과를 적용한다:
- Info-dump 해소
- Telling에서 Showing으로 변환
- 감각 묘사 보강
- 문장 리듬 조정
- 톤 일관성 확보
