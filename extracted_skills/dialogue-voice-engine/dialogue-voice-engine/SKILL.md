# Dialogue & Voice Engine (대화문 및 목소리 설계기)

판타지 소설의 대화문을 전문적으로 설계하고 작성하는 도구. 눈을 가리고 대사만 읽어도 누가 말하는지 알 수 있어야 한다는 원칙을 실현한다.

핵심 철학: 대화문은 캐릭터의 영혼이 밖으로 나오는 통로이다. 모든 대사는 그 캐릭터만이 할 수 있는 대사여야 한다. 같은 상황에서 전사와 학자와 도둑은 같은 말을 하지 않는다. 그 차이가 캐릭터의 생명이다.

---

## 전체 흐름

```
Step 1: 캐릭터별 음성 프로파일 설계 (Voice Profile)
Step 2: 서브텍스트 설계 (Subtext Design)
Step 3: 대화의 기능 점검 (Dialogue Function)
Step 4: 다종족 대화 설계 (Multi-Species Dialogue)
Step 5: 대화 태그와 비트 (Tags & Beats)
Step 6: 대화 진단 모드 (Diagnostic Mode)
Step 7: 음성 프로파일 출력 (Voice Profile Output)
```

---

## Step 1: 캐릭터별 음성 프로파일 설계 (Voice Profile)

각 주요 캐릭터에 대해 음성 프로파일을 설계한다.

### 음성 프로파일 구성 요소

어휘 수준: 학식, 계급, 직업에 따른 어휘 범위. 귀족은 추상적/수사적 어휘를, 병사는 구체적/직접적 어휘를, 학자는 전문 용어를 사용한다.

문장 길이와 구조: 짧고 끊어지는 문장은 행동적/직접적 캐릭터를, 길고 유려한 문장은 사색적/교양 있는 캐릭터를, 불완전한 문장은 긴장/흥분 상태를 나타낸다.

말버릇과 특성: 각 캐릭터에게 2-3개의 반복적 표현이나 패턴을 부여한다. 과용하면 캐리커처가 되므로 절제가 핵심이다. 예: 특정 맹세("빛의 이름으로"), 문장 시작 습관("자, 봐라"), 특유의 감탄사.

비유의 원천: 캐릭터가 사용하는 비유는 그의 경험 세계에서 나와야 한다. 대장장이는 쇠와 불의 비유를, 항해사는 바다와 바람의 비유를, 농부는 계절과 수확의 비유를 사용한다.

감정 표현 방식: 감정을 직접 말하는 캐릭터 vs 돌려 말하는 캐릭터 vs 말 대신 행동으로 표현하는 캐릭터 vs 유머로 감정을 숨기는 캐릭터.

유머 유형: 풍자적 유머, 건조한 유머, 말장난, 자기 비하 유머, 상황 유머, 유머 없음.

경칭/호칭 체계: 상대방을 부르는 방식이 관계와 권력 역학을 드러낸다. 이름 vs 직함 vs 별명 vs 경칭 vs 비칭의 사용이 관계의 친밀도와 권력 관계를 표현한다.

### 음성 프로파일 템플릿

```
캐릭터: [이름]
어휘 수준: [일상적/교양적/학술적/거친/고어체]
문장 특성: [짧고 직접적 / 길고 우회적 / 불규칙]
말버릇: [습관적 표현 2-3개]
비유 원천: [경험 분야]
감정 표현: [직접/회피/행동/유머]
유머: [유형 또는 없음]
금기어: [절대 사용하지 않는 표현]
대표 대사: [이 캐릭터를 가장 잘 보여주는 대사 1-2개]
```

## Step 2: 서브텍스트 설계 (Subtext Design)

대화에서 말하지 않는 것이 말하는 것보다 강력하다. 서브텍스트는 대화의 수면 아래에서 작동하는 진짜 의미이다.

### 서브텍스트의 유형

간접 표현: 진짜 하고 싶은 말을 직접 하지 않고 돌려 말함. "그 사람 잘 있어?"(진짜 의미: "그 사람이 그립다")

회피: 특정 주제를 피하는 패턴. 캐릭터가 특정 질문에 답하지 않거나 화제를 돌리는 것은 유령/상처(character-arc-engineer 참조)와 관련된다.

거짓말: 의도적 기만과 그것을 암시하는 미세 신호. 거짓말하는 캐릭터는 평소보다 구체적이거나(과보상), 반대로 모호해진다.

권력 역학: 누가 대화를 주도하는가, 누가 질문하고 누가 답하는가, 누가 침묵하는가, 누가 끼어드는가. 대화의 구조 자체가 관계의 권력 구조를 드러낸다.

감정의 전위: 한 감정을 다른 감정으로 표현. 슬픔을 분노로, 두려움을 공격으로, 사랑을 냉담으로 표현하는 패턴.

### 서브텍스트 설계 방법

각 주요 대화 장면에 대해:
1. 각 캐릭터가 이 대화에서 원하는 것(표면적 목표)을 결정한다
2. 각 캐릭터가 진짜로 원하는 것(심층적 욕구)을 결정한다
3. 이 두 가지의 차이가 서브텍스트가 된다
4. 서브텍스트를 대사가 아닌 행동, 침묵, 톤의 변화로 드러낸다

## Step 3: 대화의 기능 점검 (Dialogue Function)

모든 대화 장면은 최소 2가지 기능을 동시에 수행해야 한다. 단 하나의 기능만 수행하는 대화는 비효율적이다.

캐릭터 드러내기: 대사를 통해 성격, 가치관, 감정 상태를 보여준다.
갈등 진행: 대화 속에서 갈등이 에스컬레이션된다.
정보 전달: 세계관/플롯 정보를 자연스럽게 전달한다(Info-dump 대체).
관계 변화: 대화를 통해 관계가 변화한다(신뢰 형성/파괴).
분위기 설정: 장면의 톤과 긴장감을 대화로 조절한다.
복선 삽입: 대사 속에 이중 의미를 심어 복선으로 활용한다.

점검 방법: 모든 대화 장면에 대해 "이 대화가 수행하는 기능은 무엇인가?"를 확인한다. 답이 1개 이하이면 대화를 강화하거나 삭제한다.

---

## Step 4: 다종족 대화 설계 (Multi-Species Dialogue)

판타지 특유의 도전: 서로 다른 종족/문화의 캐릭터가 대화할 때.

통역/공용어: 공용어가 존재하는 경우, 비원어민이 공용어를 사용할 때의 어색함(어순 차이, 관용구 오용)을 묘사하면 종족 차이가 자연스럽게 드러난다.

문화적 오해: 같은 단어에 대한 다른 의미 부여. 예: 인간에게 "명예"는 개인의 평판이지만, 특정 종족에게 "명예"는 씨족 전체의 생존 조건일 수 있다.

비언어적 커뮤니케이션: species-culture-designer에서 설계한 커뮤니케이션 양식이 대화에 반영되어야 한다. 비인간 종족의 비언어적 신호(색 변화, 페로몬, 귀의 움직임)를 대화 비트에 포함시키면 세계관의 깊이가 드러난다.

금기어의 문화적 충돌: 한 문화에서 평범한 표현이 다른 문화에서 극도의 모욕이 되는 상황. 이것은 문화 충돌 장면에서 자연스러운 갈등 원천이 된다.

## Step 5: 대화 태그와 비트 (Tags & Beats)

### 대화 태그

기본 원칙: "said/말했다"는 투명하다. 독자가 인식하지 못하고 지나간다. 화려한 대화 태그("포효했다", "속삭이듯 중얼거렸다")는 양념처럼 아껴 쓴다.

판타지 특화: 비인간 종족의 대화 태그에 종족 특성을 반영할 수 있다("으르렁거렸다", "노래하듯 말했다", "딸깍거리는 소리와 함께 말했다").

### 대화 비트(Action Beat)

대화 태그 대신 행동 묘사로 화자를 식별하는 기법:

대화 태그 사용: "전쟁이 시작될 거야," 그가 말했다.
대화 비트 사용: "전쟁이 시작될 거야." 그는 검을 벽에 기대놓고 피 묻은 손을 닦았다.

대화 비트의 강점은 화자 식별과 동시에 캐릭터의 물리적 행동, 감정 상태, 환경 정보를 전달한다는 것이다.

### 대화 태그/비트 비율

대화 라인 3-4개마다 태그 또는 비트 1개가 적절하다. 모든 라인에 태그를 붙이면 지저분하고, 태그 없이 긴 대화가 이어지면 누가 말하는지 혼란스러워진다.

## Step 6: 대화 진단 모드 (Diagnostic Mode)

기존 원고의 대화문을 분석하고 개선점을 제시한다.

### 진단 체크리스트

| 진단 항목 | 점검 내용 | 발견 시 조치 |
|----------|----------|------------|
| 음성 균일성 | 모든 캐릭터가 같은 말투 | 음성 프로파일 차별화 |
| "As You Know" 대화 | 캐릭터가 서로 이미 아는 것을 설명 | 갈등/무지/대조 기법으로 전환 |
| 서브텍스트 부재 | 대화가 표면적 의미만 전달 | 숨겨진 동기와 감정 레이어 추가 |
| 태그 과잉 | 모든 대사에 태그 부착 | 대화 비트 활용, 필요 없는 태그 삭제 |
| 기능 부족 | 대화가 단일 기능만 수행 | 최소 2가지 기능 병행하도록 재설계 |
| 문화적 평면성 | 다른 종족/문화 캐릭터가 같은 방식으로 대화 | 종족별 언어 패턴 반영 |
| 대화 길이 불균형 | 한 캐릭터만 길게 말하고 나머지는 반응만 | 대화의 주고받음(turn-taking) 균형 |
| 감정-대사 불일치 | 캐릭터의 감정 상태와 대사 톤이 모순 | 감정 상태에 따른 어조 조정 |

## Step 7: 음성 프로파일 출력 (Voice Profile Output)

```
캐릭터 음성 프로파일 문서
━━━━━━━━━━━━━━━━

1. 캐릭터별 음성 프로파일 (각 캐릭터)
   - 어휘/문장/말버릇/비유/감정/유머/금기
   - 대표 대사 2-3개
2. 관계별 대화 톤 매트릭스
   - 캐릭터 A-B 대화의 톤, A-C 대화의 톤 등
3. 서브텍스트 가이드
   - 핵심 대화 장면의 표면/심층 의미 매핑
4. 다종족 대화 규칙
   - 공용어 사용 규칙, 문화적 금기, 비언어 신호
```

출력 포맷은 사용자 지정에 따른다.

## 스킬 연계

character-arc-engineer(캐릭터의 내면이 대사에 반영, 유령/상처가 회피 패턴을 생성), species-culture-designer(종족별 언어 패턴, 커뮤니케이션 양식), prose-style-engine(서술문과 대화문의 조화), relationship-web-mapper(관계가 대화 톤을 결정, 관계 변화가 대화 변화로 표현)
