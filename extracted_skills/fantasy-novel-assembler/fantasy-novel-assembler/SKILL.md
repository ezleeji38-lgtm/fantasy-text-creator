# Fantasy Novel Assembler (판타지 소설 조판 및 출판 준비기)

완성된 판타지 소설 원고를 전문적인 출판 형식으로 조판하고, 출판 투고에 필요한 부속 문서를 생성하는 도구.

---

## 전체 흐름

```
Step 1: 원고 구성 확인 (Manuscript Structure Check)
Step 2: 조판 실행 (Formatting Execution)
Step 3: 부속 문서 생성 (Supplementary Documents)
Step 4: 최종 패키지 출력 (Final Package Output)
```

---

## Step 1: 원고 구성 확인 (Manuscript Structure Check)

판타지 소설 원고의 표준 구성을 확인한다:

```
[전면부]
  - 속표지 (제목, 저자명, 연락처)
  - 헌정사 (선택)
  - 지도 (선택, 텍스트 설명 형태)
  - 등장인물 목록 (선택, 캐릭터가 많을 때 권장)
  - 발음 가이드 (선택, 고유 명사가 많을 때 권장)

[본문]
  - 프롤로그 (선택)
  - Part 1: [파트 제목]
    - Chapter 1: [챕터 제목 또는 번호]
    - Chapter 2: ...
  - Part 2: ...
  - ...
  - 에필로그 (선택)

[후면부]
  - 세계관 용어 사전 (Glossary) (선택)
  - 역사 연표 (선택)
  - 저자 노트 (선택)
  - 감사의 글 (선택)
  - 다음 권 미리보기 (시리즈물, 선택)
```

---

## Step 2: 조판 실행 (Formatting Execution)

원고 출력 시 반드시 /mnt/skills/public/docx/SKILL.md를 읽고 따른다.

### 조판 기본 사양

```
판형: A5 (표준 소설) 또는 B5 (대형 판타지) — 사용자 선택
폰트: 본문 11pt (Malgun Gothic / Times New Roman)
행간: 1.5배
여백: 상 2cm / 하 2cm / 좌 2.5cm / 우 2cm (제본 고려)
챕터 시작: 새 페이지, 챕터 제목 후 2-3줄 여백
파트 시작: 파트 표지 페이지 + 새 페이지에서 첫 챕터 시작
페이지 번호: 본문부터 시작, 하단 중앙
머리글: 좌측에 제목, 우측에 저자명 (선택)
```

### 투고용 표준 포맷

출판사/에이전트 투고 시 별도 포맷:
```
폰트: Times New Roman 12pt (또는 해당 출판사 지정 폰트)
행간: 2배
여백: 모든 방향 2.5cm (1인치)
페이지 번호: 우측 상단 (이름/제목 약어 포함)
들여쓰기: 각 문단 첫줄 1.27cm (0.5인치)
장면 전환: # 기호 (센터 정렬)
```

## Step 3: 부속 문서 생성 (Supplementary Documents)

### 3-1. 시놉시스 (Synopsis)

출판사/에이전트 투고용 시놉시스:

분량: 1-2페이지 (500-1,000단어), 일부 에이전트는 3-5페이지 요구
내용: 주인공 소개, 핵심 갈등, 주요 전환점, 결말(숨기지 않음). 시놉시스에서는 반전을 포함한 결말을 모두 공개한다.
톤: 3인칭, 현재형, 객관적이되 이야기의 감정적 힘이 느껴지도록.
시리즈물: 이 권의 독립적 아크를 완결하여 보여주고, 마지막 문단에서 시리즈 방향성을 1-2문장으로 간략 언급.

구조:
```
문단 1: 주인공 소개, 일상 세계, 핵심 결함/욕구
문단 2: 촉발 사건, 모험의 시작
문단 3: 핵심 갈등과 장애물, 주요 관계
문단 4: 중간점 전환, 상황 악화
문단 5: 클라이맥스, 최종 선택, 해결
문단 6 (시리즈): 잔존 질문과 다음 권 방향 (1-2문장)
```

### 3-2. 쿼리 레터 (Query Letter)

에이전트/편집자에게 보내는 투고 서신:

구조:
```
[인사 + 개인화] 1-2문장
  - 에이전트의 최근 출판물/관심사를 언급하여 왜 이 에이전트에게 보내는지 설명

[훅] 1-2문장
  - 이야기의 핵심 전제를 한 문장으로 압축

[줄거리] 150-250단어
  - 주인공(이름, 특징), 촉발 사건, 핵심 선택, 판돈
  - 결말은 공개하지 않음 (시놉시스와 다름)
  - 이야기의 감정적 핵심이 느껴져야 함

[비교 작품(Comp Titles)] 1-2문장
  - "[작품 A]의 [특정 요소]와 [작품 B]의 [특정 요소]의 교차점" 형태
  - 최근 2-5년 내 출판된 작품을 비교 대상으로 선택

[작품 정보] 1-2문장
  - 장르, 분량(단어 수), 독립작/시리즈, 완성 여부

[저자 약력] 2-3문장
  - 관련 경력/실적 (출판 이력, 관련 전문성, 수상 이력)
  - 경력이 없으면 간결하게 (억지로 늘리지 않음)

[마무리] 1문장
  - 감사 + 응답 대기
```

### 3-3. 세계관 부록 (World Appendix)

독자 참조용 세계관 부록. fantasy-worldbuilder와 magic-system-architect의 출력물에서 독자용으로 재가공:

고유 용어 사전(Glossary): 본문에 등장하는 모든 고유 용어의 간략 정의. 알파벳/가나다 순 정렬.

주요 인물 목록: 이름, 소속/종족, 1문장 설명. 본문에 이름이 등장하는 모든 인물(주요/주변).

시대 연표: 본문에 언급되는 역사적 사건의 시간순 정리. 독자가 시간 관계를 파악하는 데 도움.

마법 체계 요약: 독자 이해 수준으로 간략화한 마법 규칙. 작가용 설정 문서보다 훨씬 간결해야 한다.

## Step 4: 최종 패키지 출력 (Final Package Output)

### 출력 파일 목록

| 파일 | 설명 | 필수/선택 |
|------|------|---------|
| [제목]_manuscript.docx | 완성 원고 전체 (소설 판형) | 필수 |
| [제목]_manuscript_submission.docx | 투고용 포맷 원고 | 투고 시 필수 |
| [제목]_synopsis.docx | 시놉시스 | 투고 시 필수 |
| [제목]_query.docx | 쿼리 레터 | 투고 시 필수 |
| [제목]_appendix.docx | 세계관 부록 | 선택 |
| [제목]_series_plan.docx | 시리즈 계획 요약 | 시리즈물 선택 |

---

## 스킬 연계

모든 Phase 1-5 스킬의 결과물이 이 스킬의 입력이 된다. fantasy-worldbuilder(세계관 부록 재료), character-arc-engineer(등장인물 목록, 시놉시스 캐릭터 요약), plot-architecture-engine(시놉시스의 플롯 요약), magic-system-architect(마법 체계 부록), series-expansion-planner(시리즈 계획 문서)
